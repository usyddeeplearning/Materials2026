P21
We first have a spatial perspective to illustrate the definition of graph convolution. 

We can explicitly split the calculation into two parts. hi is the i-th node in this graph. Ni denotes the neighboring node set for the i-th node.
For the i-th node itself, we include a matrix W_0 to transform its feature hi. Also, there is another matrix W_1 to transform the features of its neighboring nodes, i.e., hj with h\in Ni.

The overall calculation could be very straightforward. You only need to introduce a weight matrix W_0 to process the node itself, and then introduce the other matrix W_1 to transform its neighboring nodes. After that, we do the summarization and apply the sigmoid activation function. 
W_1 is shared by all neighboring nodes, which is kind of weight sharing scheme in GCN. By doing so, the calculation is invariant to the permutations of nodes. Linear complexity O(E).
Here exist two different paradigms for learning on graphs, transductive and inductive. In transductive learning, the nodes and edges remain constant across the training and prediction phases. In other words, at training time, the learning algorithm has access to all the nodes and edges including those for which predictions are to be made. Note that transductive paradigm does not 
Inductive learning is that the model is first trained on a set of graphs and then applied to a separate set of graphs.

There are also limitations. This kind of convolution may not be very effective to aggregate the information. So it may be very hard to develop very deep GCNs.
And another limitation is that there is no direct use of the edge feature. You can see in the definition of the calculation, we only use the features of the graph nodes.

When we are dealing with a space, we need to define its basis. In this frequency domain, the basis can be defined through complex-valued functions, e^{2\pi i \xi t}. Given a series of complex exponentials as the basis for the frequency domain, we can write out the expression of f in the above equation. 
We have an important conclusion that this complex exponentials are actually the eigenvector of 1-d Laplace operator.
Recall that we have defined a laplacian matrix L before. This laplacian matrix is defined over the graph data. We can do the eigen decomposition for this laplacian matrix L, and then we get the eigenvector \mu, and the eigenvalue \lambda.

P32
As what we have introduced that u plays a similar role as the complex exponentials in the top formula, the Fourier transform of signal f can then be written as \hat{f}= U^{T} f. Also, we can have  \hat{h}= U^{T} h for signal h. 
Recall that U is composed of eigenvector, and thus it is a orthogonal matrix. Given \hat{f}= U^{T} f, we can get f = U\hat{f}.

f is the convolution filter. If we re-organize f into a diagonal matrix g_{\theta}, we can rewrite the top equation into UgU^{T}x.  
We need to calculate U matrix through the eigen decomposition of laplacian matrix L. 
You can see we need to optimize n parameters in this diagonal matrix, and n is the number of graph nodes. If if the graph is very large, the complexity will be higher. 

P35
We do not want eigen decomposition, as it is computational expensive. Also, we want to have some spatial localization of the graph nodes in the convolution, and want to decrease the number of parameters to optimize. 
Instead of directly optimizing theta, we assume theta can be written as a combination of eigen values, and then we only need to optimize the combination coefficients alpha. 
So we can rewrite g_theta, and derive a simplified calculation of Ug_{theta}U^{t} as the right side, where we only need to deal with the input laplacian matrix L.
Also, the number of combination coefficient k should be smaller than n, and then the optimization burden can be reduced.
